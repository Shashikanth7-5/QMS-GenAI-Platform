❓ QMS GenAI — Help & Guide
Everything you need to use this application
1
📊 Dashboard — View Quality Records
The dashboard shows all 14 quality records (Complaints, Deviations, Change Controls) loaded from the Python backend. Use filters to narrow by type, priority, or sector. Each record shows its status, priority badge, and age.
2
📋 CAPA Creation — Generate & Save CAPA Drafts
Select any record from the dropdown. Click Generate CAPA — the AI service auto-fills root cause, corrective action, preventive action, regulatory references, and effectiveness check. Edit any field, then Save Draft.
3
🔍 RCA Analysis — 5-Why & Fishbone
Select a record, choose 5-Why or Fishbone method, click Generate RCA. Then click Assess Accuracy — each step is scored on Specificity, Actionability, and Completeness. Edit weak steps and Re-assess until score is Strong, then Approve & Submit to CAPA.
4
🌳 Decision Tree — Should a CAPA be raised?
Select Complaint, Deviation, or Change Control. Answer each of the 6 gate questions Yes/No. The Python backend evaluates gates in order — the first Yes triggers a CAPA recommendation with the regulatory reference. All No = no CAPA needed.
5
🔮 Use Case 3 — Coming in Sprint 2
The third use case module is planned for Sprint 2. The backend folder structure is already in place — a new routes/usecase3.py and templates/usecase3/ are all that's needed.
